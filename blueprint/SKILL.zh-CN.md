# Blueprint · 创业蓝图

一个结构化的成功模式提取引擎。Preflight 从 2,500+ 尸检报告中问「创业怎么死」，这个 skill 从 5,000+ 个经过验证的收入故事中问「赢家到底怎么做」。

**核心命题**：成功会留下指纹。在 StarterStory（1,000+ 案例）、IndieHackers（5,000+ 收入透明的创始人）、YC Startup School 和 MicroConf 中，相同的模式反复出现。突破 $10K MRR、再突破 $100K、最终突破 $1M+ 的创始人并非运气好——他们在跑相同的招式。本 skill 将这些招式编目入库。

## 核心心智模型

**镜子，不是教条。** 每个创业都不同，但行之有效的模式是有限的。本 skill 以三种模式运行：

- **对标模式**：'我现在 $5K MRR 做 SaaS——处于我这阶段的成功者做了什么到了 $50K？' → 匹配收入阶段 + 商业模式 + 创始人类型，给出同行对标
- **差距分析模式**：'这是我的项目。我缺少哪些被验证过的成功模式？' → 5 个模块全跑一遍，标记差距，排出优先级
- **反击模式**：'Preflight 标了我的平台依赖是 🔴——成功者怎么解的？' → 交叉引用 Preflight 维度与成功者的对抗方案

## 五大模块

### 模块一：七大经典成功框架

理论基石。这些是被成功创始人反复发现的框架——经过数十年、数千家公司验证。详见 `references/success-frameworks.md`。

| # | 框架 | 核心问题 | 最适合 |
|---|---|---|---|
| 1 | **Helmer 7 种力量** | 你的可持续竞争优势是什么？ | 战略、护城河 |
| 2 | **Thiel 7 个问题** | 你在建垄断还是在竞争？ | 从 0 到 1 的创业 |
| 3 | **Graham「做不可规模化的事」** | 什么不可规模化的动作会变成你的护城河？ | $0→$1K 阶段 |
| 4 | **Mom Test（妈妈测试）** | 你是在学习还是在收集假阳性？ | 想法验证 |
| 5 | **精益 MVP 循环** | 你在建最小能教你的东西吗？ | 产品开发 |
| 6 | **Hormozi 价值方程** | 为什么有人会为这个付钱？ | 定价、定位 |
| 7 | **Rachleff PMF 测试** | 用户是在把你的产品从你手里抢过去吗？ | 增长拐点 |

> **用法**：当用户问「怎么建护城河」或「怎么验证想法」，把问题映射到对应框架，走一遍诊断。

### 模块二：收入阶段模式

每个收入里程碑上成功者实际做了什么。基于 StarterStory + IndieHackers 收入数据的交叉分析。详见 `references/stage-patterns.md`。

| 阶段 | 收入 | 核心挑战 | 致胜模式 | 卡住信号 |
|---|---|---|---|---|
| **验证** | $0 → $1K/月 | 有人付钱吗？ | 预售 + 首日收费 | '想法不错'但没有钱 |
| **PMF** | $1K → $10K/月 | 用户留下来并告诉别人吗？ | 一个渠道 + 精确定价 | 流失 > 增长 |
| **规模化** | $10K → $50K/月 | 能增长而不崩吗？ | 外拓 + 定价优化 + 向上销售 | 3 个月平台期 |
| **团队化** | $50K → $200K/月 | 能放权吗？ | 流程 + 专才 + 付费投放 | 创始人是瓶颈 |
| **引擎化** | $200K → $1M+/月 | 能复利增长吗？ | 多渠道 + PLG + 品牌 | 只在优化，没有扩展 |

**StarterStory 关键数据**：42% 突破 $10K MRR 的创始人将其首次增长拐点归因于**口碑**——不是广告、不是内容营销、不是冷启动外拓。他们做对了某件事，以至于用户主动告诉其他用户。

### 模块三：商业模式胜法

不同模式需要不同的成功打法。做市场平台的创始人和做 SaaS 的创始人的招式完全不同。详见 `references/model-patterns.md`。

| 模式 | 致胜路径 | 关键指标 | 经典陷阱 |
|---|---|---|---|
| **SaaS** | 垂直优先 → 落地扩张 | 流失<5%, LTV:CAC>3 | 过早做横向 |
| **市场平台** | 供给受限 → 一个地区 → 扩展 | 流动性比率, 抽成>15% | 贪大求全 |
| **电商/DTC** | 差异化产品 + 有机内容 | 毛利率>60%, 复购>30% | VC 补贴的获客成本 |
| **服务/代理** | 深度专精 → 产品化 → 流程规模化 | 利用率>70%, 利润率>40% | 创始人是瓶颈 |
| **内容/媒体** | 利基权威 → 社区 → 变现 | 单用户收入 | 纯广告天花板 |
| **硬件** | 预售 → 众筹 → 迭代 → DTC | 毛利率>40%, 退货率<5% | 原型 ≠ 量产 |

### 模块四：创始人招式库

可重复的、经过测试的「招式」，成功创始人反复使用。这些不是战略——是具体的、可复制的动作。详见 `references/founder-moves.md`。

| # | 招式 | 长什么样 | 数据支撑 | 适用阶段 |
|---|---|---|---|---|
| 1 | **Day 1 就收费** | 预售、咨询、候补名单押金 | 80% 年收 $100K+ 的 StarterStory 创始人 | 验证阶段 |
| 2 | **公开构建** | 在 Twitter/LinkedIn 分享收入、挣扎、决策 | 5-10x 受众增长，2-3x 付费转化 | 所有阶段 |
| 3 | **微利基统治** | 「宠物美容店 CRM」而非「CRM」 | 2x 赢单率，40% 更低流失 | PMF、规模化 |
| 4 | **阶梯式攀登** | 服务 → 产品化 → SaaS → 规模化 | 70% 更高存活率（TinySeed 数据） | 自筹资金 |
| 5 | **PLG 飞轮** | 免费层 → aha 时刻 → 病毒传播 → 升级 | 30-50% 更低获客成本（vs 销售驱动） | 规模化+ |
| 6 | **做不可规模化的事** | 人工 onboard 每一个早期用户，手写邮件 | Airbnb、Stripe、Zapier 都这么干过 | 验证阶段 |
| 7 | **空房间 Demo** | 裸原型 → 观察犹豫点 → 就做那个 | 60% 的功能是有了首批付费用户后才加的 | 验证阶段 |
| 8 | **痛苦程度阶梯定价** | 省 $10K = 收 $3K。锦上添花 = $10/月 | 始终能区分 $1M+ 和 $100K 创始人 | PMF、规模化 |

### 模块五：对标 Preflight 12 维反击

对于创业可能死的每一种方式（Preflight 的 12 个维度），成功者都有一套被验证的对抗方案。详见 `references/preflight-countermoves.md`。

| Preflight 维度 | 成功者的对抗方案 | 核心原则 |
|---|---|---|
| 1. 需求真伪 | 提取承诺，不提取意见 | Mom Test：预订单 >「听起来不错」 |
| 2. 竞争护城河 | 在规模化之前先建立 Helmer 的 1 种力量 | 后发优势 > 先发优势 |
| 3. 单位经济 | 从第一天就 LTV:CAC > 3，毛利率 > 50% | 按价值定价，不按竞品定价 |
| 4. 现金流 & 跑道 | Default-alive 计划。自筹：$10K MRR 时盈利 |「下一轮会来的」不是计划 |
| 5. 团队匹配 | 创始人-市场匹配：领域专业度 OR 痴迷级用户共情 | 独立创始人用速度补偿 |
| 6. 商业模式清晰度 | 先从一个人手里收一块钱 | 先有一个清晰的收入来源 |
| 7. 产品/技术可行性 | 上丑陋的 v1。应该让自己感到尴尬。 |「如果 v1 不尴尬，你发布太晚了」 |
| 8. 监管/法律 | 把合规建成护城河，而非障碍 |「只有我们能做」因为「我们懂规则」 |
| 9. 时机 | 抓住你不是你创造的那波浪潮 |「为什么是现在？」比「为什么？」更难回答 |
| 10. 增长与分发 | 一个渠道吃干榨净，再加第二个 | 深度 > 广度 |
| 11. 平台依赖 | Wrapper Depth Scale 第 3 级+ | 自有数据/垂直知识为护城河 |
| 12. 结构韧性 | 需求必须不依赖宏观条件而存在 |「经济衰退时还有人需要这个吗？」 |

## 如何使用

### 第一步：确定模式

询问用户——或根据问题推断：
- 「和我同阶段的别人怎么成功的？」→ **对标模式**
- 「我缺什么？」→ **差距分析模式**
- 「Preflight 标了 X 为 🔴——怎么改？」→ **反击模式**

### 第二步：收集上下文

差距分析和标杆模式需要：
- 当前收入（或阶段：想法 / 验证 / 已上线 / 增长中）
- 商业模式（SaaS / 市场平台 / 电商 / 服务 / 内容 / 硬件）
- 创始人画像（独立 / 联合创始人，技术 / 非技术，自筹 / VC）
- 当前最大挑战（一句话）

### 第三步：匹配并输出

| 模式 | 动作 |
|---|---|
| **对标** | 匹配收入阶段 + 商业模式 → 拉取阶段模式 + 模型胜法 → 输出 3-5 个最相关的同行打法 |
| **差距分析** | 针对上下文跑 5 个模块 → 标记「已有模式」vs「缺失模式」→ 排前 3 差距并给出具体招式 |
| **反击** | 映射 Preflight 维度到成功者的对抗方案 → 展示 2-3 个解决过同样问题的真实创始人案例 → 建议具体动作 |

### 第四步：输出

默认格式：结构化 Markdown 报告。

```
## 📊 Blueprint 分析

### 你的上下文
[收入阶段 / 模式 / 创始人类型 / 挑战]

### 模块一：框架匹配度
[最相关的框架 + 诊断]

### 模块二：阶段对标
| 模式 | 你 | 同行 | 差距 |
|---|---|---|---|

### 模块三：商业模式胜法
[商业模式特定的致胜招式 + 你的状态]

### 模块四：创始人招式
[按影响力排序的最高优先级招式]

### 模块五：对标 Preflight 反击
[如已跑过 Preflight：逐维度的反击方案]

### Top 3 行动
1. [最紧急、最高影响力的招式]
2. [下一个]
3. [再下一个]

### 同行参考
[2-3 个与你同阶段的真实创始人故事，来自 StarterStory/IndieHackers]
```

## 多项目对比模式

对 2-4 个项目同时跑对标模式 → 输出并列的阶段适配度和招式完成度对比。

## 与 Preflight 的配合

本 skill 设计为 Preflight 之后的**第二步**：

1. **跑 Preflight** →「你可能的死法如下。最大风险：[X, Y, Z]」
2. **跑 Blueprint（反击模式）** →「活下来的创始人是这样干的。Top 招式：[A, B, C]」

两个 skill 共享概念体系：Preflight 的 12 个维度与模块五的反击方案一一对应。使用反击模式时，始终先拉取最新的 Preflight 评分卡。

## 输出规则

- **必须有真实案例**：每条模式建议必须附 ≥1 个真实创始人故事（姓名 + 公司 + 结果），来源为 StarterStory、IndieHackers、YC 或 MicroConf
- **阶段匹配**：不要在 $0 MRR 阶段推荐 PLG。不要在 $500K MRR 需要获客渠道时推荐「Day 1 收费」
- **模式匹配**：不要对市场平台创始人推荐 SaaS 模式
- **行动优先**：每份报告必须以「本周可做的 Top 3 行动」结尾
- **语言匹配**：用户用什么语言就用什么语言输出
- **互补而非重复**：如果用户有 Preflight 数据，始终交叉引用。如果还没跑过 Preflight，建议先跑以获得完整评估
