# Data Sources & Methodology

Data sources, confidence levels, and known gaps for the GTM skill.

## Primary Data Sources

| Source | Scale | What It Provides | Used For |
|---|---|---|---|
| **StarterStory** | 1,000+ founder interviews | GTM stories with verified revenue data | All 10 channel profiles |
| **IndieHackers** | 5,000+ revenue-transparent founders | Channel experiments and stage transitions | Channel recommendations, stage matching |
| **Blueprint skill references** | 8 founder moves, 6 model playbooks | Pre-analyzed success patterns | Channel cross-referencing |
| **YC Startup School + Library** | 4,000+ YC companies | PLG best practices, growth frameworks | PLG, Community, API channels |

## Confidence Levels by Channel

| Channel | Confidence | Reason |
|---|---|---|
| Content / SEO | ⭐⭐⭐⭐⭐ | Multiple verified cases with revenue data |
| Build in Public | ⭐⭐⭐⭐⭐ | Very strong signal from IndieHackers + Twitter |
| Product-Led Growth | ⭐⭐⭐⭐ | Strong from YC + public data, less from our StarterStory set |
| Community / Audience | ⭐⭐⭐⭐ | Good from ConvertKit, The Hustle, Lenny |
| Product Hunt Launch | ⭐⭐⭐⭐ | Good data but one-time event, not a channel |
| Productized Service → SaaS | ⭐⭐⭐⭐ | Multiple cases (Basecamp, DesignJoy, Rob Walling) |
| API / Integrations | ⭐⭐⭐ | Mostly Zapier data, fewer case studies |
| Cold Email / Outbound | ⚠️ Partial | Generic benchmarks, few from our direct data |
| Partnership / Affiliate | ⚠️ Partial | Mentioned but not deeply covered in our cases |
| Paid Acquisition | ⚠️ Partial | Mostly failure stories, few successful case studies |

## Known Gaps (v0.5)

1. **Enterprise sales playbook** — Our data covers bootstrapped/SaaS up to ~$10M ARR. Enterprise sales ($50K+ ACV) requires different frameworks not covered here.
2. **Paid acquisition benchmarks** — We have more failure stories than success stories. The "avoid" advice is reliable but "when to do it" is thin.
3. **Cold email metrics by industry** — Benchmarks exist generally but not for each business model.
4. **Hardware/retail GTM** — Hardware and physical retail have very different GTM patterns not covered here (you ship product months before revenue).
5. **Non-English market GTM** — Our data is mostly English-language/internet. China, India, LATAM have different channel dynamics.
6. **Mobile app GTM** — App Store distribution, ASO, and mobile advertising are not covered.

## How to Improve v0.5 → v1.0

- Add data from GTM-specific case study sites (e.g., gtmbets.com, growthhacker.com)
- Crawl IndieHackers for GTM-specific posts (250+ tagged posts exist)
- Add benchmarks from public SaaS data sets (e.g., OpenView PLG benchmarks)
- Add paid acquisition benchmarks from real SaaS data (if available)
