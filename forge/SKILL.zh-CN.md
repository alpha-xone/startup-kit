# Forge · 想法生成引擎

> **在写一行代码之前，找到真正的问题。**

Forge 是 preflight 的 G0 前置工具。Preflight 问「这个想法行不行」，Forge 问「什值得做」——在你还没想法的时候。

## 核心理念

**最好的创业想法来自你亲身经历或观察到的问题——不是从头脑风暴「好点子」中诞生。** Forge 通过结构化的发现方法，帮你系统性地发现和验证真实痛点。

### 三种模式：

- **痛点挖掘**：「我想在 [某领域] 做点东西——有什么问题值得解决？」→ 从行业数据、用户抱怨、竞品空白中提取痛点和证据
- **快速排雷**：「我有一个想法——禁得起推敲吗？」→ 5 个问题快速过滤
- **相邻机会**：「我懂 [某领域]——有什么相邻问题可以解决？」→ 从 preflight 的 16 行业数据中映射痛点

## Forge 框架

### 第一步：选择来源

选一个或组合：

- **个人痛点** — 你最近被什么烦到了？什么手工流程你在重复做？
- **行业空白** — Preflight 的 16 行业数据告诉你哪里死得最多→反向就是机会
- **抱怨挖掘** — Reddit、应用商店评论、客服论坛。找「要是有个工具能……就好了」
- **工作流观察** — 观察别人工作。什么太慢？什么需要绕路？
- **组合拳** — 一次做多个小东西。总有一个会发芽（Marc Lou 一年 17 个产品法）

### 第二步：快速过滤

每个候选想法打 0-2 分：

| 问题 | 0 分 | 1 分 | 2 分 |
|---|---|---|---|
| 你亲身经历这个痛点吗？ | 没有 | 间接 | 是的，自己有 |
| 你能说出 5 个有同样痛点的人吗？ | 不能 | 大概可以 | 能，有名有姓 |
| 已经有人在为解决方案付费了吗？ | 没有 | 有免费替代 | 有付费产品 |
| 这个痛点在变严重吗？ | 稳定 | 缓慢增长 | 快速增长 |
| 你愿意付费解决它吗？ | 不愿意 | 可能 $10/月 | 是的，$50+/月 |

**≥6 分**：有潜力 → 进入 preflight G1
**3-5 分**：有意思但需更多验证 → 15 次 Mom Test 对话
**<3 分**：还不够 → 继续挖掘

### 第三步：输出

```
## 🔨 Forge：机会报告

### 领域：[用户输入的领域]

### 发现的痛点
1. [痛点] — 证据: [来源] — 得分: [X/10]
2. [痛点] — 证据: [来源] — 得分: [X/10]

### 最佳机会
[最高分想法 + 原因]

### 下一步
[进入 preflight G1 / 更多 Mom Test / 继续挖掘]

### 参考
[行业数据洞察、竞品空白等]
```

## 输出规则

- **痛点为先**：没有真实痛苦证据就没有「好点子」
- **人，不是市场**：「5 个有确切痛苦的人」>「这是个 X 亿的市场」
- **诚实打分**：低分 = 继续挖，不是「说服自己」
- **配合 preflight**：Forge 的输出喂给 preflight G1，不要重复 G1 的深度
