# Preflight · 创业体征检查

一个把「模糊想法 → 有证据支撑的立项决策」全过程结构化的评估系统。融合创业墓园数据（2,500+ 尸检报告）、行业热力图（16×12 矩阵）、Big Tech 产品墓园（307 个 Google 死亡产品）。

## 核心心智模型

**这是一个漏斗 + 一个循环。**

- **漏斗（fail-fast）**：五道闸门 G0→G4，越往后越贵。一个在 G1 就该死的 idea，不放它到 G3 浪费时间。
- **循环（loop）**：每次跑完把结果写入记分卡。复查时读上一轮 → 算增量 → 检查止损线 → 更新决策。

**G2 是本系统的核心**——12 个统一维度覆盖所有项目类型。每个维度的评估深度根据项目特征自动调整：

| 项目特征 | 触发的深度诊断 |
|---|---|
| 依赖第三方 API/平台（AI 模型、Shopify、AWS、iOS 等） | 维度 11「平台依赖性」深入：套壳厚度、替代风险、成本结构 |
| 硬件/制造业 | 维度 7「产品可行性」深入：供应链、量产、良率 |
| 金融/电信/医疗 | 维度 8「监管合规」深入：牌照、资本金、消费者保护 |
| 市场平台 | 维度 10「增长路径」深入：冷启动、供需平衡 |
| 政策/叙事驱动型 | 维度 12「结构韧性」深入：周期位置、补贴依赖 |
| B2C 消费品 | 维度 3「单位经济」深入：LTV/CAC、复购率、退货率 |
| 纯软件/SaaS | 无特殊深入——走标准诊断即可 |

> **核心原则**：12 个维度对**所有项目**必修——因为墓园数据告诉我们，一个 SaaS 公司和一家硬件工厂可能死于同一把刀（例如竞争、团队、现金）。但每个维度的**追问深度**由项目的具体特征决定——不要对一家面包店问「你的 API 成本占比多少」。

## 五道闸门

| 闸门 | 名称 | 回答的问题 | 淘汰标准 |
|---|---|---|---|
| G0 | 发掘 Discover | 有没有真实痛点值得做？ | 找不到≥1 个有原生抱怨证据的痛点 |
| G1 | 需求验证 Validate | 这个痛点是真的、够大、有人肯付钱吗？ | 需求分 < 阈值 |
| G2 | 斩杀线压测 Stress-test | 12 维度统一诊断：会掉进创业墓园吗？ | 综合斩杀风险 🔴 且无逃脱路径 |
| G3 | 立项自检 Self-check | 时机对吗？我适合做吗？ | 任一「一票否决项」踩中 |
| G4 | 决策+止损 Decide | GO / 迭代 / KILL？ | 输出决策 + kill criteria |

## G2 统一 12 维度斩杀线

### 设计原则

1. **12 个维度对所有项目必修**——每个维度都有核心检查问题
2. **追问深度按项目特征调节**——只在匹配时加载 `references/conditional-diagnostics.md` 中对应的深度问题
3. **所有维度共享同一数据源**——墓园数据（CB Insights、Loot Drop、LOOTR、Killed by Google）不为 AI 和非 AI 分别开两个诊断分支
4. **Big Tech 墓园启示在诊断完成后统一给出**——不分项目类型

### 维度速查表

| # | 维度 | 墓园数据 | 核心问题（所有项目） | 何时深入追问 |
|---|---|---|---|---|
| 1 | **需求真实性** | CB Insights #2 (42%) | 谁在为这个问题付费？没有你他们现在怎么解决？ | 项目是「先有技术再找场景」时——追问是否陷入了解决方案主义 |
| 2 | **竞争可防御性** | CB Insights #1 (53%), 覆盖所有行业 | 大厂/已有玩家能不能在一个 sprint 内复制你？用户迁移成本是多少？ | 当行业 = 信息科技/通信服务/消费品（最高频竞争死亡行业）— 追问大厂射程 + 替代品威胁 |
| 3 | **单位经济** | LOOTR 消费品 #2 (114 例), 总体 18% | 每单赚钱吗？LTV > 3×CAC？规模扩大后边际成本降还是升？ | B2C 消费品 → 追问复购率/退货率。依赖第三方 API → 追问 API 成本占比 + 降价假设。硬件 → 追问物料清单(BOM) + 制造良率 |
| 4 | **现金流与跑道** | CB Insights #3 (29%), 消费品 62 例 | 现有资金撑几个月？跑道 < 12 个月 🟡，< 6 个月 🔴。有没有止损线？ | 资本密集型（硬件/制造/零售）→ 追问库存周转 + 应付/应收 gap |
| 5 | **团队适配度** | CB Insights #4 (20-23%) | 团队有没有人真正理解目标用户的领域？技术人员+商务人员是否平衡？ | 技术团队做行业应用 → 追问认知错配风险。一人创业 → 追问产品+市场覆盖 |
| 6 | **商业模式清晰度** | 总体 10% | 用户为什么愿意付钱而不是用免费的？有几个收入来源？ | 先免费再变现 → 追问转化路径。只有一种收费方式 → 追问定价天花板 |
| 7 | **产品/技术可行性** | 总体 15%, 硬件 #1 | 能造出来吗？瓶颈在技术还是制造？ | 硬件 → 追问制造/供应链/量产。深度技术 → 追问原型→中试→量产路径。软件 → 标准诊断 |
| 8 | **监管合规** | 总体 12%, 金融 22 例, 电信 25 例 | 法律会不会拦你？需要牌照吗？客户资金在你手上吗？ | 金融/电信/医疗 → **必修深度追问**（牌照、资本金、消费者保护、跨境合规） |
| 9 | **时机判断** | 总体 8% | 太早了（用户需改变行为）还是太晚了（已有不可撼动的玩家）？ | 项目需要用户改变习惯 → 追问「早了 20 年」风险（Webvan 模式） |
| 10 | **增长与分发** | 通用 | 前 1000 个用户从哪来？有传播机制吗？不花钱能找到用户吗？ | 市场平台 → 追问冷启动（鸡和蛋）。B2C → 追问 CAC + 渠道。B2B → 追问销售周期 + 企业采购流程 |
| 11 | **平台依赖性** | 76% AI 项目死于薄套壳；可推广至所有平台依赖业务 | 你的生意有多大比例建立在别人的平台上？如果平台改规则/涨价/封号，你能活吗？ | 依赖第三方 API（AI 模型、支付、地图等）→ 追问套壳厚度 5 阶。依赖单一渠道（iOS/Shopify/AWS）→ 追问替代成本 |
| 12 | **结构韧性** | 中国历史周期：个体户→下海→双创→灵活就业→一人公司 | 你的项目是独立市场需求驱动，还是被政策/叙事裹挟？如果补贴/风口消失，需求还在吗？ | 项目出现在政策/叙事之后（一人公司、超级个体、灵活就业、碳中和补贴等）→ 追问周期位置 + 补贴依赖度 |

### 维度 1：需求真实性

**墓园数据**：CB Insights #2 杀手，42% 的初创企业死于无市场需求。高发行业：通信服务 (80)、信息科技 (46)。

**标准检查**（所有项目）：
- 用户现在是怎样解决这个问题的？
- 这个问题有多痛？「难受但能忍」vs「不解决活不下去」？
- 有没有人已经为类似的解决方案付过钱？
- 「他们现在怎么解决？」——如果答案是「不用解决」或「Google Sheets」，需求为假

**深入追问**（当项目「先有技术/能力再找场景」时触发）：
- 你的产品是「先有痛点再用技术优化」还是「有了技术才想出来的」？
- 你的效率提升是否建立在一个客户不信任的市场？（医疗、法律、金融等强信任领域 — 参考 Legion 案例：效率高≠客户信任）
- 5 个愿意付费的种子用户在哪？去和他们聊——不要猜

**真实案例**：Juicero ($120M, $699 WiFi 榨汁机→手挤更快)、Magic Leap ($3.5B, 14 年后死亡)、Google Stadia (玩家不需要云游戏)

### 维度 2：竞争可防御性

**墓园数据**：CB Insights #1 杀手，53%。LOOTR 热力图覆盖所有行业。信息科技 (260)、通信服务 (257)、消费品 (178)。

**标准检查**（所有项目）：
- 核心功能是否是大厂/已有玩家的 RdMap 上的一个 checkbox？
- 进入壁垒是什么？（不只是「更好的 UI」）
- 用户迁移成本是多少？< 5 分钟 = 🔴

**深入追问**（信息科技/通信服务/消费品 — 最高频竞争死亡行业）：
- 大厂做同样的功能边际成本是多少？（集成进现有产品几乎免费 = 🔴）
- 大厂不做这个功能是因为**看不上**还是**做不了**？（做不了 = 你的护城河）
- 如果你成功了他们会注意到你吗？（越晚越好）
- 竞争生态位判断：大厂核心战场 → 🔴 / 大厂边缘场景 → 🟡 / 大厂不愿做的 → 🟢

**真实案例**：Vine (2 亿月活→被 Instagram 碾压)、Quibi ($17.5 亿)、Clubhouse ($10 亿→3 个月归零)、Neeva (无广告搜索→用户不搬家)

**逃脱路径**：找大厂不愿做的生态位（隐私敏感、高定制、线下集成、合规牌照）；建立网络效应/数据飞轮；先 niche 赢再扩展

### 维度 3：单位经济

**墓园数据**：LOOTR 消费品 #2 杀手 (114 例)。典型「消费者陷阱」—— CAC > LTV。

**标准检查**（所有项目）：
- 每单赚钱吗？不补贴的情况下毛利是多少？
- LTV / CAC ≥ 3 倍？
- 规模扩大后边际成本降还是升？

**深入追问 — 按项目特征**：
- **B2C 消费品**：复购率？退货率？DTC 还是平台？——注意「VC 补贴获客→促销结束用户消失」的经典陷阱
- **依赖第三方 API**：API 成本占收入比例？如果底层涨价 2 倍还能活吗？毛利率是否 > 50%？（参考：AI wrapper 50-60% vs 传统 SaaS 80-90%。低于 50% = 🔴）
- **硬件**：物料清单 (BOM) 成本？制造良率？库存周转天数？
- **市场平台**：补贴供/需哪一端？补贴停止后交易量是否维持？

**真实案例**：大量 DTC 品牌死于「用 VC 的钱补贴 CAC → 忠实用户在融资停止后消失」

### 维度 4：现金流与跑道

**墓园数据**：CB Insights #3 (29%)。中国小微企业：25% 能撑 6 个月现金流，30%+ 一个月都撑不住。

**标准检查**（所有项目）：
- 现有资金能撑几个月？< 12 个月 🟡，< 6 个月 🔴
- 有止损线吗？（「在 X 月内达不到 Y 指标就关停」）
- 「下一轮会到」是唯一现金流策略吗？→ 🔴

**深入追问 — 资本密集型**（硬件/制造/零售）：
- 库存周转天数？应付账款 vs 应收账款 gap？
- 前期投入中多少是沉没成本（模具、厂房、装修）？

**逃脱路径**：从第一天收钱。YC 经验：等到「规模效应」才开始收钱的公司 90% 活不到那天。

### 维度 5：团队适配度

**墓园数据**：CB Insights #4 (20-23%)。85% AI 项目死于「认知错配」——技术人员不懂领域、领域专家不懂技术。但这不是 AI 特有——任何技术驱动的行业应用都有这个风险。

**标准检查**（所有项目）：
- 团队中是否有人真正理解目标用户的领域？（不是在书上读的）
- 技术能力和商务能力是否平衡？
- 创始人见过目标用户吗？——没见过的 = 🔴

**深入追问**：
- 技术团队做行业应用 → 有没有领域专家搭档？参考 Zillow Offers（损失 $5 亿因为算法说房子值钱但没人闻过甲醛）
- 一人创业 → 产品+市场两边都覆盖了吗？
- 第一次创业 → 失败率显著高于有经验的（70% 一人公司创始人是首次创业）

### 维度 6：商业模式清晰度

**墓园数据**：总体 10%。Vine (2 亿月活→无变现→死)。

**标准检查**（所有项目）：
- 用户为什么愿意付钱而不是用免费替代品？
- 有几个收入来源？（1 个 = 🔴, 2 个 = 🟡, ≥3 个 = 🟢）
- 「先有流量再想怎么变现」= 🔴

**深入追问**：
- 先免费再变现 → 免费→付费的转化路径画出来了吗？有数据验证过吗？
- 收费方式是否匹配用户心智？（例：消费者对 SaaS 订阅反感 vs 企业对 consumption-based 接受）

### 维度 7：产品/技术可行性

**墓园数据**：总体 15%，但这是硬件 #1 死因。注意：排第六而非第一 —— **初创企业不是死于不会造产品，而是死于不会卖。**

**标准检查**（所有项目）：能造出来吗？瓶颈在哪？

**深入追问**：
- **硬件**：Demo ≠ 量产。完成小批量试产了吗（100-1000 个）？实地验证过供应链每个环节吗？——硬件创业者的关键能力不是设计，是制造项目管理
- **深度技术**（新材料、新药、新工艺）：原型→中试→量产路径明确吗？每个阶段的时间和资金需求？
- **纯软件**：标准诊断即可——技术风险通常不是主要死因

### 维度 8：监管合规

**墓园数据**：总体 12%，但金融 (22) 和电信 (25) 是致命级。

**标准检查**（所有项目）：法律会不会拦你？

**深入追问 — 仅金融/电信/医疗**（这些行业触发深度诊断）：
- 需要牌照吗？资本金要求？
- 客户资金在你手上吗？→ 消费者保护法 > 你的创新
- 加密/跨境：注册地决定了谁保护你的客户——也决定了你会不会坐牢
- 参考：FTX ($32B)、Celsius ($4.7B)——监管失控不是次要问题，是致命一击

> 如果你不在金融/电信/医疗，标准检查即可。

### 维度 9：时机判断

**墓园数据**：总体 8%。「太早了」比「太晚了」杀的多。

**标准检查**（所有项目）：
- 用户需要改变行为习惯吗？→ 你在「太早了」象限
- 已有不可撼动的玩家吗？→ 你在「太晚了」象限

**真实案例**：Webvan ($1.2B, 1999→2001) 是对的——早了 20 年。Google Wave (2010) 早了 5 年——Slack/Notion 后来赢了同一个市场。

### 维度 10：增长与分发

**标准检查**（所有项目）：
- 前 1000 个用户从哪来？（具体渠道，不是「做营销」）
- 产品有传播机制吗？
- 不花钱能找到用户吗？

**深入追问 — 按项目特征**：
- **市场平台**：冷启动策略——先供后需还是先需后供？补贴哪一端？补贴停了交易量还在吗？——如果 3 个月内没有自然增长（不依赖补贴的新增），你在烧钱而非建平台
- **B2C**：CAC？主要获客渠道？复购/留存率？
- **B2B**：销售周期？谁签支票？（用户≠买家）——企业采购流程是隐形杀手

**真实案例**：Neeva ($77.5M 仍无法突破冷启动)

### 维度 11：平台依赖性

**墓园数据**：76% AI 项目死于薄套壳——但这背后的「平台依赖」逻辑适用于所有依赖单一平台/API 的业务。

**标准检查**（所有项目）：
- 你的生意有多大比例建立在别人的平台上？（API、应用商店、电商平台、云服务商）
- 如果平台改规则/涨价/封号/自建同样功能 → 你能活吗？

**深入追问 — 触发条件：项目依赖第三方 API 或平台**：
- **套壳深度 5 阶**（原创自 Monica 联合创始人 Suki，从 AI wrapper 推广至所有平台依赖型业务）：
  1. 直接调 API/用平台基础功能 → 卷 UI/价格 = 🔴
  2. 组合多个 API/平台功能 → 卷设计和体验 = 🟡
  3. 嵌入自有数据/内容/垂直知识 → 卷领域深度 = 🟢
  4. 自我强化型数据飞轮 → 用户越多产品越好 = 🟢
  5. 自建核心基础设施 → 卷研发能力 = 🟢
- 你的利润中有多少被平台/API 成本吃掉？
- 切换平台/API 的成本？（技术迁移 + 用户影响）

**真实案例**：无数 Shopify 插件/AI wrapper/iOS 独占 app 被平台一次规则变更或功能内建抹去

### 维度 12：结构韧性

**墓园数据**：来自中国市场深度分析——创业/自由职业有清晰的**历史周期规律**。每次就业承压→政府号召「自己养自己」→一个新叙事诞生。

**中国创业周期**：个体户(1979) → 下海(1992) → 大众创业/双创(2014) → 地摊经济/灵活就业(2020) → 一人公司/超级个体(2025)

**标准检查**（所有项目）：
- 你的项目是独立市场需求驱动，还是被政策/风口/叙事裹挟？
- 如果补贴/风口/媒体报道消失，需求还在吗？

**深入追问 — 触发条件：项目出现在某个政策/叙事之后**：
- 你是因为「XX 是风口」还是因为「这是一个真实问题」而创业？
- 抛开「超级个体」「一人公司」「AI 创业」这些概念，你的商业模式本质是什么？（个体户？自由职业？小生意？还是真正的 scalable business？）
- 用新工具给自己改个名字（「超级个体」）并不能改变商业本质——效率提升 ≠ 商业模式升级
- **核心判断**：你的项目在经济上行、就业充分的时候，还有人需要吗？

**70% 一人公司创始人是首次创业**——第一次创业的失败率远高于有经验的。
