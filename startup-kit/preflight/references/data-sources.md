# 数据源详解 · Data Sources & Methodology

## 核心数据源

### Loot Drop (loot-drop.io)

- **规模**: 1,749 家失败初创企业，$535.4B 烧毁的 VC 资金
- **数据结构**: 每家公司包含：名称、赛道、成立-死亡日期、融资金额、失败原因分类、重建方案（Rebuild Plan）
- **分类体系**: 22 个失败分类、16 个行业
- **特色功能**: 搜索（支持 typo）、筛选、Dashboard、Top 10 死亡排行榜、Rebuild Plans（从失败中提取的商业机会）
- **更新频率**: 每周二/五添加新案例
- **重建方案包含**: 做什么 → 市场分析 → 构建步骤 → 技术栈建议 → 收入模式
- **技术栈**: Next.js + Vercel

### Failory (failory.com)

- **主墓园**: 200+ 初创企业/产品失败分析
- **子墓园**: 
  - Startup Cemetery: 120+ 初创企业
  - Google Cemetery: 100+ Google 产品
  - Amazon Cemetery: 50+ Amazon 产品
- **分析深度**: 每篇包含成立背景、融资情况、死因分析、经验教训
- **订阅量**: 40,000+ 创始人
- **技术栈**: Next.js + Cloudflare

### CB Insights 初创企业尸检数据集

- **在线版本**: 483 份尸检报告 (cbinsights.com/research/startup-failure-post-mortem/)
- **Kaggle 版本**: 890 条记录 (1992–2024)，包含 7 个行业子表
- **SQL 分析结果** (来自 Zakariyahaa 的项目):
  - 890 家失败初创企业
  - 信息行业占 42%
  - 平均存活 8 年
  - 竞争杀死 53%，无预算 43%，停滞 21%，趋势转变 13%，无法变现 12%
  - 融资与存活时间的相关系数 +19.8%
  - 无预算与竞争负相关 -36.2%（说明这两个很少同时出现——公司要么死于竞争，要么死于没钱，但通常不是两者同时）

### LOOTR 热力图 (lootr.io/graveyard)

- **矩阵维度**: 16 个行业 × 12 个失败原因
- **核心发现**:
  - 竞争是 #1 杀手，覆盖所有行业
  - 三个行业占所有失败的三分之二：通信服务 (447)、消费品 (417)、信息科技 (406)
  - 消费品陷阱：第二大杀手是单位经济不行 (114 例)
  - 「无市场需求」在通信服务 (80) 和信息科技 (46) 最常见
  - 联合创始人冲突异常地弱（在整个热力图中属于最低频的死亡信号）
  - 监管/法律主要只在金融 (22) 和电信 (25) 行业显现
  - 「产品/技术本身不行」并非主要死因——公司不死于不会造产品，死于不会卖产品

### Killed by Google (killedbygoogle.com)

- **总计**: 307 个产品
- **构成**: 服务 215 (70%)，应用 68 (22%)，硬件 24 (8%)
- **时间范围**: 2006–至今
- **分类**: Apps、Services、Hardware
- **常见死因模式**:
  1. 策略性收购后整合失效 → 关停
  2. 内部 sponsor 离职 → 团队解散 → 产品被抛弃
  3. 免费产品无变现 → 无经济独立生存能力 → 被合并
  4. 实验项目 → 没达到内部指标 → 砍掉
  5. 被内部新产品替代（自我蚕食）
- **技术栈**: Next.js + Vercel，静态数据

### Product Grave (productgrave.net)

- **规模**: 99 个死亡科技产品
- **特色**: 像素艺术墓园风格、AI 神谕（预验尸分析）、统计面板
- **覆盖**: Magic Leap ($3.5B)、Clubhouse、Neeva、Google Stadia、FTX、Google Hangouts、Celsius Network 等
- **技术栈**: Next.js + Vercel + Cloudflare

### Unbiased Ventures (unbiasedventures.ch)

- **范围**: 2024–2025 年重大失败案例
- **分析框架**: 7 个 VC 评分维度映射
- **案例**: Northvolt、Theranos、FTX、Builder.ai、Canoo、Lilium 等

## 数据局限性

1. **幸存者偏差的反面**: 墓园只收录已知的失败——大量「从未被人知道的失败」没有被记录
2. **自报偏差**: 尸检报告由创始人撰写 → 可能低估团队/人际问题，高估外部因素
3. **VC 后验偏差**: 很多分析是在失败后做的，归因可能有「事后诸葛亮」效应
4. **中美差异**: 大部分数据来自美国 VC 生态 → 中国创业者需要根据本土情况调整（如监管环境、融资节奏、用户行为）
5. **AI 热潮偏差**: 2019 年后涌入大量 AI 相关失败 → 可能偏离其他行业的长期趋势
