# G3 · 立项自检 Self-check

需求验证（G1）和斩杀线压测（G2）解决的是「这事该不该做、会不会死」。真正让项目死掉的，往往是下面这些 G2 12 维度不能完全覆盖的辅助维度。逐项打分，并检查一票否决项。

## 四层自检

### 一、时机（Why now）
同一个点子早两年是进坟场、晚两年是红海。必须能清晰回答：**是什么变化让「现在」成了对的时间点？**
- 新技术（AI 让成本骤降 / 能力刚够用）
- 监管变化
- 用户行为迁移

没有清晰的 why-now → 多半是伪机会或时机未到。评分：why-now 越具体、越近期，分越高。

### 二、自身契合（Founder-market fit）
你在这个领域有没有**不公平优势**？
- 领域知识 / 现成受众 / 独特视角 / 已有分发渠道。
没有这层优势，你就是在跟一堆同等条件的人拼纯执行力。评分：优势越硬、越难被复制，分越高。

### 三、机会成本
做这个 = 不做什么？把它和用户**当前的主线**放一起比。
- （对本用户）Borki 投资是主线，任何 side project 都在抢注意力带宽——除非这个 side project 能反哺主线，否则机会成本很高。
评分：占用的核心资源越多、对主线拖累越大，分越低。

### 四、分发冷启动（最被低估）
独立开发失败，多数是**分发失败**而非产品失败。开工前必须有具体路径：
- **首批 100 个用户从哪来？** SEO / 某个你已经在的社群 / build in public / 别的具体渠道。
- 「我到时候发个推」不算路径。
名言：**先想清楚怎么分发，再决定做什么。** 评分：路径越具体可执行，分越高。

## 一票否决项（踩中任一 → 直接 KILL 或强制暂停）

### 合规 / 法律红线
数据隐私、IP、牌照逐一过一遍。**重点：预测 / 押注类产品**（如 cup 世界杯预测）——一旦涉及真金押注或被认定为博彩性质，各地监管差异极大（香港、内地、及目标用户所在地各不相同），直接决定：产品形态、能不能收钱、能否上架应用商店。**这条必须在写第一行代码前问清楚。**

### 平台单点依赖
是否把整个产品架在某一家的 API / 政策 / 流量上？平台一改规则就归零。

### 单位经济为负（AI 产品尤其致命）
每次调用的 API 成本 vs 定价，规模化后毛利是否为正。「用户很喜欢但每个用户都让你亏钱」是平台依赖型项目进坟场的常见死法。（此项与 G2 维度 3 和维度 11 重叠，若已跑 G2 可引用其结论，不重复评。）

## 精简版斩杀线（仅当无法加载完整 G2 12 维度时的降级方案）

正常情况下 G2 使用 SKILL.md 中的完整 12 维度诊断。若环境限制无法加载完整维度、又来不及查阅 reference，用下面 9 问做一次快速自查，每条给 🟢/🟡/🔴：

1. **需求**：真实痛点还是自我想象？（有没有 G1 的付费证据）
2. **套壳**：你的价值层有多厚？薄套壳 76% 会死，API 成本吃掉 15-30% 收入。
3. **经济**：单位经济成立吗？AI wrapper 毛利 50-60% vs 传统 SaaS 80-90%。
4. **盈利**：商业模式可持续吗？只会卖会员/积分 ≈ 必死。
5. **竞争**：在大厂射程内吗？75% 输给科技巨头。
6. **增长**：能突破冷启动吗？（见上面的分发一节）
7. **团队**：认知有没有错配？85% 团队失败死于认知错配。
8. **现金流**：钱能撑到 PMF 吗？30-38% 死于现金流耗尽。
9. **结构性泡沫**：是顺势而为还是被风口裹挟？
