## 触发条件：依赖第三方 API 或平台

适用于：AI/LLM wrapper、Shopify 插件、iOS 独占 app、AWS-only 架构、Stripe/Plaid 依赖型 fintech、任何将核心能力外包给第三方 API 的项目。

### 维度 11「平台依赖性」深度追问

**套壳深度 5 阶**（原创来自 Monica 联合创始人 Suki，从 AI wrapper 推广至所有平台依赖型业务）：

1. 直接调 API / 用平台基础功能 → 卷 UI/价格 = 🔴
   - 你的价值完全由底层提供商定义
   - 用户切换到你没有任何成本 → 你实际上是在帮 API 提供商获客
2. 组合多个 API / 平台功能 → 卷设计和体验 = 🟡
   - 有价值但脆弱——任一依赖方改定价/规则都会冲击你
3. 嵌入自有数据 / 内容 / 垂直知识 → 卷领域深度 = 🟢
   - 用户留下来是因为你的数据/知识——不只是 API
4. 自我强化型数据飞轮 → 用户越多产品越好 = 🟢
   - 用户使用产生的数据让产品更智能/更个性化 → 切换成本递增
5. 自建核心基础设施 → 卷研发能力 = 🟢
   - 你拥有了自己的技术栈——不被任何单一提供商绑架

**追问**：
- 你的利润中有多少被平台/API 成本吃掉？> 30% = 🔴
- 如果底层 API/平台涨价 2 倍，你能活吗？
- 切换 API/平台的成本？（技术迁移 + 用户影响 + 数据可移植性）
- 有没有至少一个备选方案/冗余？（多云、多平台）

**AI 特有数据**：
- AI 墓地：76% 项目死于薄套壳
- AI wrapper 毛利率 50-60%（vs 传统 SaaS 80-90%），快速增长时甚至 25%
- API 成本吃掉 15-30% 收入
- 李开复 TC-PMF：Technology Cost × Product-Market Fit——推理成本是「移动目标」

### 维度 3「单位经济」深度追问（平台依赖版本）

- 每笔交易的平台/API 成本 vs 收入占比？
- 毛利率是否 > 50%？（低于 50% = 🔴，因为平台依赖业务的毛利率天然低于独立业务）
- 随着用户增长，平台/API 成本是线性增长还是递减？（如果线性→规模无益）
- 免费用户的平台/API 成本由谁承担？
- 商业模式是否建立在「平台/API 会降价」的假设上？
- 实施分层策略：简单查询用小模型/缓存/rule-based，复杂查询用大模型/API

### 维度 2「竞争可防御性」深度追问（平台依赖版本）

**大厂射程分析**：
- 大厂能不能免费提供你的核心功能来增加主产品黏性？→ 能 = 🔴
- 你是否依赖大厂的平台/API/生态？（政策变化会直接杀死你）
- 如果你成功了他们会注意到你吗？（越晚越好）
- 75% 失败项目输给科技巨头，71% 输给直接竞争

**竞争生态位**：
- 大厂核心战场（搜索、社交、办公、电商）→ 🔴
- 大厂边缘场景（小众垂直、特定工作流）→ 🟡
- 大厂不愿做/做不好的场景（隐私敏感、高定制、线下集成）→ 🟢

**洞察**：正因为平台/API 工具普适化——如果你只有一个想法，没有任何护城河。真问题：大厂不做这个功能是因为看不上，还是做不了？

## 触发条件：硬件 / 制造业

### 维度 7「产品/技术可行性」深度追问

- Demo ≠ 量产。在融大钱前完成小批量生产（100-1000 个）了吗？
- 亲自身在工厂管供应链——不能靠邮件和电话
- 制造良率？物料清单(BOM)成本？库存周转天数？
- 供应链每个环节实地验证过吗？（一级供应商、二级供应商、物流、质检）
- 关键零部件的供应集中度？（只有一个供应商 = 🔴）
- 开模具/生产线的沉没成本？——一旦投入无法回收

### 维度 4「现金流与跑道」深度追问（硬件版本）

- 库存积压吃现金——周转天数？
- 应付账款 vs 应收账款 gap？——被供应商催款但客户还没付款 = 经典流动性危机
- 前期投入中多少是沉没成本？——花了的钱要不回来

## 触发条件：金融 / 电信 / 医疗（受监管行业）

### 维度 8「监管合规」深度追问

**致命级**——不是次要问题。如果在这些行业，合规必须在写第一行代码前解决。

- 需要牌照吗？牌照获取周期和成本？
- 资本金要求？
- 客户资金在你手上吗？→ 消费者保护法 > 你的创新
- 是否涉及跨境？（注册地管辖权 + 客户所在地法律 = 双重合规）
- 加密/DeFi：注册在「监管天堂」≠ 安全——危机爆发时，注册地决定了谁保护你的客户（也决定了你会不会坐牢）

**真实案例**：FTX ($32B→几周归零)、Celsius (冻结 $4.7B)、多家加密/借贷平台在 2022-2023 监管清扫中倒下

### 维度 1「需求真实性」深度追问（强信任领域版本）

- 你的效率提升是否建立在一个客户根本不信任你的市场上？
- 参考 Legion（法律 AI）：效率再高也打动不了客户因为「不想把身家性命交给一个不透明的大模型」——**效率高 ≠ 客户信任**
- 医疗：患者数据隐私、诊断准确性、FDA/CE 审批
- 法律：律师-客户特权、责任归属、不透明性问题
- 金融：KYC/AML、客户资金保护、系统性风险

## 触发条件：市场平台

### 维度 10「增长与分发」深度追问（市场平台版本）

**冷启动的死亡循环**：两边都在等对方先到 → 都不够多 → 烧钱补贴一边 → 钱烧完

- 先供后需还是先需后供？选哪一边先「赢」？
- 补贴哪一端？补贴停了交易量还在吗？
- **如果在 3 个月内没有看到自然增长（不依赖补贴的新增），你不是在建平台——是在烧钱**
- 先集中在一个极小的地理区域（一个城市/商圈/社区）做到供需双充足
- 不要用 VC 的钱补贴——用更好的工具/服务/机制让其中一方自然而然地来

## 触发条件：政策 / 叙事驱动型

### 维度 12「结构韧性」深度追问

**中国创业/个体经济历史周期**：
- 1979 — 个体户：800 万知青返城，政策放开自谋生路
- 1992 — 下海：市场化改革，主动离开体制/被迫下岗
- 2014 — 大众创业/双创：经济低增长，「以创业促进就业」
- 2020 — 地摊经济/灵活就业：疫情冲击，2 亿人「灵活就业」
- 2025 — 一人公司/超级个体：青年失业率高企，「用 AI 改变时代的新势力」

**核心洞察**：当社会无法提供足够多体面的工作岗位时，就会号召大家「自己养自己」。每一次都会走出几个成功者（幸存者偏差），但对 99.99% 的人是充当金字塔垫脚石。

**追问**：
- 你的项目是独立市场需求驱动，还是被政策/社会叙事裹挟？
- 如果没有任何政府补贴/政策红利，需求还存在吗？
- 你是因为「XX 是风口」还是因为「这是真实问题」而创业？
- 抛开「超级个体」「一人公司」「AI 创业」这些概念，你的商业模式本质是什么？（个体户？自由职业？小生意？真实的 scalable business？）
- 用新工具给自己改个名字（「超级个体」）并不能改变商业本质——效率提升 ≠ 商业模式升级
- **核心判断**：你的项目在经济上行、就业充分的时候，还有人需要吗？

**高风险信号**：
- 「AI 创业」作为核心卖点而非手段
- 项目出现在某个政策/社会叙事之后（一人公司、超级个体、灵活就业）
- 找不到 3 个非政策驱动、非叙事驱动的真实付费客户
- 70% 一人公司创始人是首次创业——失败率显著更高

## 触发条件：B2C 消费品

### 维度 3「单位经济」深度追问（消费品版本）

**「消费者陷阱」**：增长快、死得更快。CAC > LTV 的经典模式。

- 物理商品毛利率 20-40%（vs 软件 70-90%）
- 线上获客成本 $30-100/人——如果在 90 天内不能让一个用户回本，这个用户是亏的
- 复购率是北方星指标——不是首单获客成本
- 库存压力：滞销库存吃现金、过季库存卖不出、退货率侵蚀毛利
- 定价天花板：消费品无法像 SaaS「10x 更好 = 10x 价格」——市场锚定
- DTC 陷阱：VC 补贴获客 → 「忠实用户」在促销结束后消失
- 大厂挤压：Amazon 自有品牌对标爆款、大品牌跟风复制
- 小范围测试（pop-up、限时售卖、小批量）→ 验证后扩产

**墓园数据**：LOOTR 消费品 #2 杀手 114 例

## 触发条件：一人创业

### 维度 5「团队适配度」深度追问（一人创业版本）

- 产品+市场两边都覆盖了吗？——一人同时做产品开发和用户探索 = 必然有一个被忽略
- 有没有外部专家/顾问作为「虚拟合伙人」？
- 第一次创业 → 失败率显著高于有经验的创业者（70% 一人公司创始人是首次创业）
- 是否有「最少需要第二个人」的节点？——识别并在那之前找到搭档
