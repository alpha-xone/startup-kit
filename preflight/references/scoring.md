# 评分口径 · 阈值 · 决策 Scoring

打分不是为了精确，是为了**逼出证据、并在循环里可比较**。每个分数背后必须有一条能指出来的证据，不能凭感觉。

## 维度评分标尺（0-5）

统一含义：
- **0-1** 反面证据 / 缺失：有证据表明这一维度是弱点，或完全没有任何证据支撑。
- **2** 弱：有想法但缺硬证据。
- **3** 及格线：有一条像样证据支撑。
- **4** 强：多条独立证据互相印证。
- **5** 极强：证据充分且难以被竞争者复制。

各维度要点回顾：
| 维度 | 打分看什么 | 出处 |
|---|---|---|
| 需求 | 真实性 + 规模 + 付费意愿三类证据 | G1 / validate.md |
| 斩杀线 | ai-graveyard 整体风险（🟢/🟡/🔴 折算 4/2.5/1） | G2 |
| 时机 why-now | 变化是否具体、近期 | G3 |
| 自身契合 | 不公平优势是否硬、难复制 | G3 |
| 分发 | 首批 100 用户路径是否具体可执行 | G3 |
| 机会成本 | 对主线的拖累（越拖越低分） | G3 |
| 合规 | 过 / 踩雷（非 0-5，是一票否决） | G3 |

## 闸门通过阈值

- **G1 需求分 ≥ 3** 才进 G2；否则退回 G0 或 KILL。
- **G2** 若 ai-graveyard 判「整体斩杀风险 🔴 且无逃脱路径」→ KILL；🔴 但有明确逃脱路径 → 迭代；🟡/🟢 → 进 G3。
- **G3** 任一「一票否决项」踩中（合规红线 / 平台单点依赖致命 / 单位经济确定为负）→ KILL 或强制暂停。

## G4 综合决策

先算一个**加权综合分**（权重可按项目调整，默认如下），再叠加否决项：

```
综合分 = 需求×0.30 + 斩杀线×0.25 + 分发×0.20 + 自身契合×0.15 + 时机×0.10
（机会成本作为整体下调因子：拖累大则综合分 ×0.8）
```

决策映射：
- **🟢 GO**：综合分 ≥ 3.5，且无一票否决。→ 推进，设下一里程碑 + 到期日。
- **🟡 迭代**：综合分 2.5–3.5，或某单一维度是明显短板但可补。→ 给出「具体改什么 + 要补什么证据」，下一轮 loop 复查。
- **🔴 KILL**：综合分 < 2.5，或踩中任一否决项且无解。→ 说清死因 + 可复用资产。

> 阈值是默认值，不是圣旨。可根据用户风险偏好上下调，但**调整前先声明**，别偷偷改。

## Kill Criteria（止损线）写法

GO 或迭代的项目，**必须**在记分卡里写下可证伪、带时限的止损线。好的止损线长这样：
- 「落地页上线后 30 天内组织渠道拿不到 50 注册 → KILL」
- 「首批 20 个访谈里 <3 人愿意预付 → 换方向」
- 「6 周内单位经济做不到毛利为正 → KILL」

反例（不可用，因为无法证伪）：「如果感觉没起色就停」。

止损线的意义是**在你冷静时替未来那个上头的你做决定**。循环模式每轮必须逐条核对这些线，触发即给 KILL 建议，并直白对抗沉没成本。
