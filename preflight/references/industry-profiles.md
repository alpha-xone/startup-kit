## 细分行业深度分析

### 消费品 — 最典型的死亡方程式

**死亡方程式：** 增长快 + 单位经济差 + VC 补贴 + 钱烧光 = 💀

**为什么消费品这么难：**
1. **低毛利 + 高 CAC**: 物理商品毛利率 20-40%，而线上获客成本 $30-100/人——如果在 90 天内无法让一个用户回本，这个用户就是亏的
2. **DTC 陷阱**: 大量 DTC 品牌用 VC 的钱补贴获客 → 「忠实用户」在促销结束后消失
3. **库存压力**: 滞销库存吃现金、过季库存卖不出、退货率侵蚀毛利
4. **定价天花板**: 消费品定价受市场锚定——无法像 SaaS 一样「10x 更好就 10x 价格」
5. **大厂挤压**: Amazon 自有品牌直接对标爆款、大品牌跟风复制

**如何存活：**
- 先做小范围测试（pop-up、限时售卖、小批量）再砸钱扩产
- 建立自有渠道（DTC 官网、微信私域）降低平台佣金
- 让复购率成为第一优先指标（不是首单获客成本）
- 找有忠诚度的 niche → 小但赚钱 > 大但亏钱

### 通信服务（社交/通讯） — 赢家通吃的死亡竞技场

**死亡方程式：** 产品无壁垒 + 大厂复制 + 用户无迁移成本 = 💀

**为什么通信服务这么难：**
1. **网络效应是双刃剑**: 好消息——用户多了产品才值钱；坏消息——在用户多之前没人来。Clubhouse 在 3 个月内从 $10 亿估值归零就是因为网络效应从未真正启动
2. **复制门槛极低**: 任何一个通信/社交功能都可以被已有平台在一个 sprint 内复制并免费推出
3. **用户忠诚度极低**: 在社交/通讯上，用户宁愿留在已经有朋友的平台，不愿为了「稍微好用一点」搬家

**如何存活：**
- 不要做「另一个 [已有大平台] 的更好版本」——先找到他们忽略的 sub-niche
- 考虑 B2B 通信工具而非 B2C 社交（企业付费 > 消费者付费）
- 如果做社交，先在一个极小的社区验证网络效应（10 万人的社群比 1000 万泛用户都有价值）

### 市场平台 — 冷启动的死亡循环

**死亡方程式：** 先有鸡还是先有蛋 → 两边都不够多 → 烧钱补贴一边 → 钱烧完 = 💀

**为什么市场平台这么难：**
1. 需要同时说服供应端和需求端加入——每一方都在等对方先到
2. 补贴供应端（卖家/司机/房东）→ 成本爆炸 → 钱烧完
3. 无法同时补贴两端 → 必须选一边先「赢」→ 选择失误 = 满盘皆输

**如何存活：**
- 先集中在一个极小的地理区域（一个城市/一个商圈/一个社区）做到供需双充足
- 不要用 VC 的钱补贴供需——用更好的工具/服务/机制让其中一方自然而然地来
- 阶段判断：如果你在 3 个月内没有看到自然增长（不依赖补贴的新增），你的平台不是个平台——是个烧钱机器

### 硬件 — 技术风险杀死你

**死亡方程式：** 技术看起来可行 + 量产不可能 + 供应链断裂 + 钱烧完 = 💀

**为什么硬件这么难：**
1. **demo ≠ 量产**: 在实验室里成功制造一个原型 vs 在工厂里每周量产 10,000 个——是两个世界
2. **资金需求巨大**: 光是开模具就可能花 $100K-$1M，而很多硬件创业者在融到生产轮之前就已经没钱
3. **供应链是隐形的护城河**: 别人能买到比你更便宜的零件、更好的良率、更快的交货——不是因为技术，是因为关系/规模/经验

**如何存活：**
- 在融大钱之前至少完成小批量生产（100-1000 个）并发现所有制造陷阱
- **亲自身在工厂**——不要靠邮件和电话管供应链
- 硬件创业者的关键能力不是设计能力，是制造项目管理能力

### 金融科技 — 监管是隐形杀手

**死亡方程式：** 技术对了 + 市场需求对了 + 忽略监管 = 💀

**为什么 fintech 这么难：**
1. 你在做金融的时候，监管机构把你当银行管——但你没有银行级别的合规团队
2. 加密特别危险：2022-2023 的清算中几乎所有大型破产（FTX、Celsius、BlockFi）都有监管/合规问题
3. 消费者保护法 > 你的创新——如果客户资金有风险，监管机构根本不会听你的创新故事

**如何存活：**
- 在写第一行代码之前，先找到律师验证你能不能在目标市场合法运营
- 金融行业的竞争优势之一就是合规——做得越好、壁垒越高
- 不要注册在「监管天堂」自以为安全——当危机爆发时，你的注册地决定了谁保护你的客户（也决定了你会不会坐牢）

## 行业生存指数（估算）

基于 LOOTR 数据中「总失败数 / 行业创业总数」的粗略估算：

| 存活难度 | 行业 | 原因 |
|---|---|---|
| 🔴 极难 | 通信服务、消费品、信息科技 | 进入壁垒最低 → 最多尸体 |
| 🟡 困难 | 金融、食品、零售、游戏 | 低毛利 + 强竞争 + 特定行业门槛 |
| 🟡/🟢 中等 | 制造、物流、教育 | 中高进入壁垒 → 竞争较少但资本需求大 |
| 🟢 较易 | 医疗器械、B2B 企业软件 | 高技术壁垒 → 相对少的竞争，但技术风险替代了竞争风险 |

> **注意**: 「较易」≠「容易」。医疗器械可能是因为公司死于技术/监管/资金问题，而非竞争——死亡率可能同样高，只是死因不同。
