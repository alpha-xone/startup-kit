# G0 · 需求发掘 Discover

目标：找到**至少 1 个有原生抱怨证据的真实痛点**。真需求藏在吐槽、求助、「有没有工具能……」里，不在问卷的客套回答里。核心动作是「去用户抱怨的地方，而不是客套的地方」。

## 挖掘句式（跨平台通用）

在下列渠道搜索这几类句式，命中即是信号：
- 「我希望有个工具能……」/ "I wish there was a tool that..."
- 「有没有软件可以……」/ "is there software for..."
- 「我现在花钱雇人做……」/ "I'm paying someone to..."
- 「好难用」「求推荐」「求同款」「至今没有好用的」

**判据**：同一个痛点出现 ≥10 次、且没人给出满意方案 → 候选机会。按时间排序能抓到还没被工具覆盖的新兴痛点。

## 渠道地图

### 海外 C 端 / 通用
- **Reddit**：r/SaaS、r/Entrepreneur、r/smallbusiness + 对应垂直兴趣子版块。含金量最高。
- **Hacker News**：Ask HN（需求求助）、Show HN（已有方案 + 评论区真实反馈）。
- **Product Hunt**：看评论区「我更想要 X」「没解决 Y」，不是看首页。
- **Indie Hackers**：创业者互相拆解真痛点 vs 伪需求。
- **X (Twitter)**：搜同样句式，最新鲜、未被覆盖的痛点。
- **Discord / Slack 垂直社群**：日常闲聊暴露的问题比调研真实。

### B 端 / SaaS（付费意愿最强）
- **G2 / Capterra**：看竞品 2-3 星差评——用户已在付钱说明需求真，差评说明现有方案烂，就是切口。
- **LinkedIn**：行业人群的职业痛点。
- **YC Request for Startups**：YC 每隔几月发一次，「YC 在要 = 有需求」的市场路标。

### 国内（贴近中文场景）
- **V2EX**：国内开发者/独立开发聚集地，「求推荐」「有没有」帖。
- **即刻**：产品/独立开发/创业者密度高。
- **小红书**：C 端需求最前线（生活/消费/女性向），搜「有没有」「求推荐」「好难用」。
- **知乎**：高赞问题下的追问、「至今没有好用工具」类评论。
- **少数派**：工具类需求 + 高付费意愿人群。
- **抖音 / B 站评论区**：反复出现的「求同款」「怎么才能……」。

## 自动化工具（从手动翻帖升级）
- **GummySearch / SparkToro**：自动化 Reddit / 受众研究，按时间排序抓新兴痛点。
- **专门的痛点挖掘 AI 工具**（PainBase 一类）：聚合 Reddit/X/PH 信号并打分。省时间，但**必须回源核对原帖**，AI 摘要易丢上下文。
- **国内**：5118、百度指数、微信指数、巨量算数——看需求词的搜索/内容热度曲线。

## G0 产出
一张候选痛点清单，每个痛点附：
1. 一句话痛点描述（尽量用用户的原话）
2. ≥1 条原生证据来源链接（帖子/差评/评论）
3. 初步的目标人群

产出后进入 G1 逐个验证。若一个痛点连一条像样的原生证据都找不到 → 别放进漏斗。
